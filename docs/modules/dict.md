# 字典管理

::: warning 注意
字典管理模块正在开发中，目前仅提供基础页面框架。完整功能将在后续版本中提供。
:::

## 功能概述

字典管理用于管理系统中的各种枚举数据，如状态值、类型值等，提供统一的数据字典管理功能。

## 功能规划

- ✅ 字典类型的增删改查
- ✅ 字典项的增删改查
- ✅ 字典数据的缓存管理
- ✅ 字典数据的导入导出

## 预期字段

### 字典类型字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 字典名称 | string | ✅ | 字典显示名称 |
| 字典类型 | string | ✅ | 字典唯一标识 |
| 状态 | number | ✅ | 字典状态 |
| 备注 | string | ❌ | 字典描述 |

### 字典项字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 字典标签 | string | ✅ | 字典项显示名称 |
| 字典键值 | string | ✅ | 字典项值 |
| 字典排序 | number | ❌ | 显示排序 |
| 状态 | number | ✅ | 字典项状态 |

## 预期功能

### 字典类型管理

- 字典类型的 CRUD 操作
- 字典类型状态控制

### 字典项管理

- 同一类型下字典项的管理
- 字典项排序调整
- 字典项状态控制

### 数据缓存

- 字典数据的内存缓存
- 缓存更新机制

## 使用场景

字典管理常用于：

- 用户状态（启用/禁用）
- 性别选项（男/女）
- 订单状态（待支付/已支付/已取消）
- 优先级（高/中/低）
